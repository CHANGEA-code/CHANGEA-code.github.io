Java笔记

一.语言基础

1.一旦创建了 String 对象，那它的值就无法改变了

理解：值无法改变指的是一开始String指向的内存空间的值不再改变。

若更改String的值的话，它会重新创建一块空间，舍弃原来的空间。

  • 要想对字符串修改的话，需要用StringBuffer 和 StringBuilder 类。

2.StringBuffer和StringBuilder：

  • 由于 StringBuilder 相较于 StringBuffer 有速度优势，所以多数情况下建议使用 StringBuilder 类。然而在应用程序要求线程安全的情况下，则必须使用 StringBuffer 类。

9.==和equals的区别:

  • ==比较两个基本类型时,比较的是两个类型的值;比较两个引用类型时,比较的时两个类型的地址.

11.单例模式的两种写法：饿汉式和懒汉式：

  • 什么是单例模式：只创建一个对象，减少系统的开销。

  • 代码写法：

  • ~~~java
    package com.java.demo04;

    public class SingleTest {

    public static void main(String[] args) {
    Instance ins1 = Instance.getInstance();
    Instance ins2 = Instance.getInstance();
    System.out.println(ins1 == ins2);//true
}

    }

    class Instance{

    //私有构造函数，不让外部创建该类的对象
private Instance(){
}
//懒汉式写法
//内部创建一个对象，作为属性，也是唯一一个对象。
private static Instance instance = null;
public static Instance getInstance(){
    if(instance == null){
         synchronized(Instance.class){            //线程安全写法
            if(instance == null){
            instance = new Instance();
            }
        }
    }
    return instance; 
}
---------------------------------------------------

    return instance;

    }

接口和抽象类的异同：

  • 同：二者都不能实例化，都需要类去继承或者实现才能实例化。

14.异常的分类：

  • 分两类：编译时异常(非RuntimeException)，如IOException等；运行时异常(RuntimeException)，如数组越界，错误强制类型转换等。

  • 编译时异常指编写程序的时候会报错。而运行时异常是程序运行时才报错

  • 异常处理的方式：

    • try-catch-finally: catch捕获多个异常时，要注意异常的子父类关系，父类写在子类在下面。

    • throw + 异常类型: 只是抛出异常，没有真正的处理异常。

    • 应用场合：try-catch-finally是将异常处理掉了，用于调用的方法父类没有throws方式处理异常，就只能用这种方式。

      throw + 异常类型 用于父类的方法中有使用throws方式处理异常。

  • throws和throw的区别：

    • throws：为异常处理，也可理解为抓异常。
    • throw：为抛出一个异常对象。

17.sleep()和wait()的异同：

  • 同：二者都能使得当前运行的线程阻塞。
  • 异：
    • sleep():可以在任何需要的地方调用；wait()：只能用在同步代码块或同步方法中。
    • 是否会释放同步监视器(锁)： 如果二者都使用在同步代码块或同步方法中时，sleep()不会释放该监视器，wait()则会释放该监视器。

18.List中的ArrayList,LinkedList,Vector的比较：

  • ArrayList：线程不安全，因此效率较高，使用频率较高。结构是Object[]数组。

  • LinkedList：结构是双向链表，对于插入，删除的操作效率比ArrayList高。

  • Vector：List接口古老的实现类；线程安全，因此效率较低。结构是Object[]数组。

  • 相同点：存储数据的特点相同：存储有序的，可重复的数据。

19.map

  • HashMap的底层实现原理?（具体看Day07代码MapTest.java）
    • JDK7及以前：底层使用数组+链表；首先new的时候会创建长度16的Entry[] table数组。存储时，对key求哈希值，根据值去找存储位置；若该位置有内容了，则比较二者哈希值；若相同，再比较内容（即调用key所在类的equals去比较），若还相同，则将value值替换老的value值；若不同，则添加。涉及到扩容时，扩大为原来的2倍。
    • JDK8：底层使用数组+链表+红黑树；与JDK7及以前不同之处：
      • 1.new的时候没有创建一个长度为16的数组，put()被调用时才创建；
      • 2.底层数组为Node[]类型。
      • 3.当数组某个索引位置上元素以链表形式存储的数据的个数>8并且当前数组的长度>64时，将该索引位置上的数据改为以红黑树结构存储。(提高查找效率)
    • Map结构的理解：
      • Map中的key：无序，不可重复的。用Set来存储所有的key；因此key所在的类要重写equals()和hashCode()方法（以HashMap为例）
      • value：无需，可重复。用Collection来存储所有的value；value所在的类要重写equals()方法，判断是否为空等作用。
      • entry：（一个键值对就构成了一个Entry对象）无序，不可重复。用Set来存储所有的entry。
  • HashMap和Hashtable的异同？
    • 前者线程不安全，效率高，但可以保存null值的键值对；
    • 后者线程安全，效率低，但不可以保存null值的键值对。
  • CurrentHashMap与Hashtable的异同？未解决

22.对象序列化机制：

  • 将Java对象转换成与平台无关的二进制流，从而可以将这种二进制流持久地保存在磁盘上，或通过网络将这种二进制流传输到另一个网络节点。

  • 反序列化机制：将这种二进制流转换成Java对象。

  • 对象流中，用序列化ObjectOutputStream类保存。反序列化用ObjectInputStream类保存。

25.父类和子类的转换：

  • 向上转型：子类转成父类，也就是多态的实现，直接子类赋值给父类，不需要什么限制，直接赋值即可。
  • 向下转型：有限制，需要实现了多态的父类，通过强制转换才能转换成子类。

1.调用statement时容易产生sql注入问题：

如何避免？用PreparedStatement替换。因为PreparedStatement是预编译的语句，句式的结构已经定下了，不会更改。

2.preparedStatement和statement的联系和区别：

  • 联系：二者都是sum公司的jdk定义的一套规范，都是接口。其中preparedStatement是statement的子接口。

  • 区别：

    • statement存在的弊端：1.存在拼串，繁琐。2存在sql注入问题。
    • preparedStatement就是用来解决statement存在的问题。其优点有：能预编译sql语句，缓存了sql语句，每次通过占位符传入参数，就不用总是进行数据库检验。
