2.找出后继节点

题目：给定一个新的二叉树节点类型：比一般的二叉树多一个指向父节点的指针。如下：

public class Node {

public int value;

public Node left , right , parent;

public Node(int val) { value = val; }

}

要求在二叉树中找到一个节点的后继节点

什么是后继节点和前驱节点？

  • 中序遍历中，node的下一个节点叫作node的后继节点，node的上一个节点叫作node的前驱节点。

思路：对每个节点，判断是否有右子树：

  • 若有，则后继节点是右子树的最左节点。

  • 若没有，则往上符合“我是父亲的左节点”的节点，若有则返回该父节点，否则继续往上找。若父节点为null，（说明要找的节点是中序遍历的最后一个节点）返回null。

口诀：判右子树，有找最左子，无找右上父。

3.序列化和反序列化(熟悉)

序列化是指将一颗二叉树转化为字符串的形式。一般转化为“节点值_”的形式。

反序列化是指将字符串转化为一颗二叉树。一般是以‘_’分割成字符串数组，再放到队列中，遍历构造二叉树。

以上可以用前中后层序的方式。

4.折纸问题

题目：请把一段纸条竖着放在桌子上，然后从纸条的下边向上方对折1次，压出折痕后 展开。 此时折痕是凹下去的，即折痕突起的方向指向纸条的背面。 如果从纸条的下边向上方连续对折2次，压出折痕后展开，此时有三条折痕，从 上到下依次是下折痕、下折痕和上折痕。 给定一个输入参数N，代表纸条都从下边向上方连续对折N次。 请从上到下打印所有折痕的方向。 例如:N=1时，打印: down N=2时，打印: down down

思路：

假想一颗二叉树，左子树是凹，右子树是凸，对深度为n的二叉树进行中序遍历，得到的结果即为所求。

口诀：想二叉树，左子为凹，右子为凸，中序遍历。

5.二叉树节点中的最大距离

题目：从二叉树的节点a出发，可以向上或者向下走，但沿途的节点只能经过一次，到达节点b时路 径上的节点个数叫作a到b的距离，那么二叉树任何两个节点之间都有距离，求整棵树上的最大距离。

思路：

对每个节点，分两种情况：

  • 不参与，则最大距离可能是左子树的最大距离或右子树的最大距离。

  • 参与，则最大距离可能是左子树高度+1+右子树高度。

每个节点的最大距离对三者取max即可。

套用模板解法：每个节点都要从左右子树中拿到最大距离和高度，自己也要计算出这些信息往上返回。

6.最大快乐值

题目：派对的最大快乐值 员工信息的定义如下:

class Employee {

public int happy; // 这名员工可以带来的快乐值

List subordinates; // 这名员工有哪些直接下级

} 公司的每个员工都符合 Employee 类的描述。

整个公司的人员结构可以看作是一棵标准的、 没有环的 多叉树。树的头节点是公司唯一的老板。除老板之外的每个员工都有唯一的直接上级。 叶节点是没有 任何下属的基层员工(subordinates列表为空)，除基层员工外，每个员工都有一个或多个直接下级。 这个公司现在要办party，你可以决定哪些员工来，哪些员工不来。

但是要遵循如下规则。

1.如果某个员工来了，那么这个员工的所有直接下级都不能来

2.派对的整体快乐值是所有到场员工快乐值的累加

3.你的目标是让派对的整体快乐值尽量大 给定一棵多叉树的头节点boss，请返回派对的最大快乐值。

思路：对每一个节点，判断是否参与：

  • 若参与，最大快乐值为该节点的所有子节点不参与时，子节点包含的树构成的最大快乐值相加

  • 若不参与，最大快乐值为子节点参与或者不参与时，子节点包含的树的最大快乐值取max，再和所有的子节点得出的值相加

最后对根节点的以上两种情况取max即可。

模板思路：每个节点要从子节点得到子节点参与时的最大快乐值和不参与时的最大快乐值。
